### 提交规范
提交格式：
```
<type>(<scope>): <version>: <subject>
// 空一行
(<body>)
// 空一行
(<footer>)
```

其中 `<type>` 为以下七种：
```
feat：    新功能（feature）
fix：     修补bug
docs：    文档（documentation）
style：   格式（不影响代码运行的变动）
refactor：重构（即不是新增功能，也不是修改bug的代码变动）
test：    增加测试
chore：   构建过程或辅助工具的变动
```
